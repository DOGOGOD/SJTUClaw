# Course Report Skill

## 能力范围

此 skill 用于生成大学课程相关的 Markdown 报告草稿，覆盖以下类型：
- 课程小论文 / 期末论文
- 课程学习总结
- 课程实验报告
- 读书报告
- 思政课心得报告

## 工作流程

1. **理解需求**：从用户任务中提取以下关键信息：
   - 报告类型（论文 / 总结 / 实验报告 / 读书报告）
   - 课程名称
   - 主题或标题
   - 字数要求（如未指定，默认 1500-2000 字）
   - 参考材料（workspace 中的文件路径或用户直接在消息中给出的材料摘要）
   - 保存路径（必须指定，用于通过 overwrite_file 写入 workspace）

2. **收集材料**：
   - 如果用户提供了 workspace 中的参考文件路径，使用 read_file 读取
   - 如果用户直接在消息中描述了材料，直接使用
   - 如果材料不足，主动向用户说明并请求补充

3. **生成草稿**：
   - 标准结构（可根据报告类型调整）：

   ```
   # 标题

   ## 摘要 / 引言
   （简要说明报告背景和目的，100-200 字）

   ## 正文
   （根据主题展开，分 2-4 个小节，每个小节有明确论点）

   ## 结论 / 总结
   （总结核心观点，提出个人思考或展望）

   ## 参考资料
   （列出引用的文献、材料、课堂内容等）
   ```

   - 语言风格：学术但不生硬，逻辑清晰，避免空泛套话
   - 遵守字数要求

4. **保存草稿**：
   - 使用 **overwrite_file** tool 将生成的 Markdown 内容写入用户指定的保存路径
   - 写入操作会经过已有 approval 流程，用户确认后才实际写入
   - 写入完成后告知用户文件已保存

5. **后续修改**：
   - 如果用户对草稿不满意，可以使用 edit_file 进行局部修改
   - 鼓励用户提出具体的修改意见

## 注意事项

- 生成的是**草稿**而非最终稿，用户需要审阅和修改
- 如果引用了具体数据或事实，尽量标注来源
- 不使用过于口语化的表达
- 不编造不存在的事实或数据
