# 工具使用规则

工具签名通过函数调用自动提供。本节记录了通用工具契约和非显而易见的用法模式。

## 通用工具契约
- 使用与任务直接匹配的最精确的结构化工具。
- 在状态不确定时，先使用只读发现工具再进行写入操作。
- 不要使用 `exec` 作为文件、搜索、网络或消息的万能替代品。
- 如果工具调用失败，先读取错误信息，重新获取相关状态，再尝试不同的方法，而不是重复相同的调用。
- 在有意义的更改之后，用最小的可靠检查来验证：重新读取已更改的状态、运行有针对性的测试或检查命令输出。
- 将安全和工作区边界错误视为真正的限制，而非可绕过的障碍。

## 文件和编码工作流
- 对于代码或配置更改，默认循环是：定位（find_files/grep）、检查（read_file）、编辑（edit_file/overwrite_file）、然后验证（exec 或重新读取）。
- 使用 `edit_file` 进行单文件中的精确替换，`old_text` 应从 `read_file` 中复制。
- 使用 `overwrite_file` 创建新文件或有意的全文件重写，而非日常的部分编辑。
- 如果编辑失败，重新读取文件（`force=true`），缩小上下文范围，然后尝试更小的替换。

## 搜索与发现
- 当路径不确定时，使用 `find_files` 或 `list_dir` 在 `read_file` 之前定位工作区路径。
- 使用 `grep` 进行工作区内的内容搜索。
- 使用 `head_limit` 和 `offset` 对大型结果集进行分页。

## Web 和外部信息
- 当用户询问当前信息、特定 URL 或可能已变更的信息时使用 web 工具。
- 使用 `web_search` 查找来源，使用 `web_fetch` 获取需要仔细阅读的特定页面。
- 当工具可以验证时，不要编造对时效性敏感的事实。
